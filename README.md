# My Portfolio - Rewritten in Svelte

## Deployed on Fly.io

This is the current README of my portfolio. Below are the tech stacks used:

### Frontend:
- Svelte
- Vite
- Tailwind CSS
- PostCSS
- Autoprefixer
- Prettier with Prettier Plugin for Svelte
- ESLint with ESLint Plugin for Svelte
- TypeScript

### Backend:
- Better SQLite3
- SQLite3
- dotenv
- Marked

### Dev Tools:
- Svelte-Check
- TypeScript ESLint Plugin
- TypeScript
- ESLint
- Prettier
- Tailwind CSS

### Build & Deployment:
- Docker
- Fly.io

### Build Scripts:
- `dev`: Run Vite development server
- `build`: Build project using Vite
- `preview`: Preview project using Vite
- `check`: Run Svelte-Kit sync and Svelte check with TypeScript
- `check:watch`: Run Svelte-Kit sync and watch for changes with Svelte check and TypeScript
- `lint`: Check code formatting with Prettier and linting with ESLint
- `format`: Format code using Prettier

### Dependencies:
- Svelte Kit
- Svelte Adapter Node

### Dev Dependencies:
- @flydotio/dockerfile
- @sveltejs/adapter-auto
- @sveltejs/kit
- @types/better-sqlite3
- @types/fs-extra
- @typescript-eslint/eslint-plugin
- @typescript-eslint/parser
- autoprefixer
- eslint
- eslint-config-prettier
- eslint-plugin-svelte
- postcss
- prettier
- prettier-plugin-svelte
- svelte
- svelte-check
- tailwindcss
- tslib
- typescript

### Additional Info:
- The project is set up to use TypeScript for type checking and linting.
- Docker is utilized for containerization.
- The project is deployed on Fly.io.
